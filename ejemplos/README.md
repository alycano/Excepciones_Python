Para armar este proyecto, lo primero que hice fue separar la lógica de lo que 'queremos que pase' de lo que 'podría salir mal'. Me enfoqué en usar el bloque try para la operación principal, como la división o el manejo de archivos, y luego fui añadiendo los except específicos para que el programa no se estallara si metía un dato raro o un cero. Lo más bacano fue aplicar el finally, porque así me aseguro de que el programa siempre me avise cuando termina el proceso, sin importar si hubo error o no. Al final, lo organicé todo en carpetas para que el código quedara limpio y fácil de leer, tal cual como lo venimos trabajando en los laboratorios del SENA.
