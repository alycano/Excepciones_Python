Para resolver este reto, me enfoqué en crear una función que fuera capaz de anticiparse a los errores típicos que comete un usuario al ingresar datos. Lo primero que hice fue meter la captura de datos y la operación matemática dentro de un bloque try, porque sabía que al convertir las entradas a enteros con int() o al hacer la división, el programa podía fallar. Implementé dos except específicos: uno para el ValueError, que me sirve para avisar cuando el usuario mete letras en vez de números, y otro para el ZeroDivisionError, para controlar el caso de la división por cero. Lo más importante fue el uso del finally, ya que con este bloque logré que el mensaje 'Operación finalizada' aparezca siempre, dándole un cierre limpio al proceso sin importar si la división salió bien o si saltó algún error. Con esto, el programa no se detiene de golpe y le da una respuesta clara al usuario, que es lo que buscamos en el desarrollo de software.
